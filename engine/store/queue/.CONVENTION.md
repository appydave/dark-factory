# Ticket-first (standing rule, David-ratified 2026-07-04)
EVERY unit of work gets a ticket JSON in this queue — even when a builder
subagent executes it instead of the engine. Builder-executed tickets carry
"executor": "builder-agent" and move queue/ -> running/ -> done/ like any
other, so the audit trail, digest, and queue are the single source of truth.
App IDEAS are different: those go to brains/dark-factory/app-pipeline/.

## External-research tickets (added 2026-07-04)

"kind": "external-research" / "executor": "external-research-agent" tickets are
self-contained research questions for David's friend-operated online agent (NOT
a Claude Code worker, no file access, not automated by orchestrator.py). They
carry "question" + "context" (context is ALWAYS inlined literal text — the
consumer can't read files) + "output_schema" instead of a code-editing "prompt".

⚠️ **orchestrator.py does NOT filter tickets by "kind"** — `queue_files()` grabs
every `.json` in `queue/`. Do not run the pool while external-research tickets
sit in `queue/`, or it will try (and fail, harmlessly but wastefully) to dispatch
them to a tmux Claude worker. Move/claim these manually per
`backlog/2026-07-04-research-agent-queue-handover.md`.

No `audit.jsonl` entry required for these (no worker/session_id to log) — the
ticket's own queue/running/done movement plus its `results/<ticket>.json` file
is the audit trail.
