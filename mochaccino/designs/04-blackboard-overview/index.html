BThe Three Worker Contracts

Write-Store-Directly
Default · all transform steps in probes 1–4
Worker writes the EAV record itself, returns a one-line ack. Orchestrator never sees the payload. Max isolation. The Oscar fix in practice.
Conductor-Writes
Checkpoints · seeds · manifests
Conductor itself writes (no subagent). Used when the conductor needs to mediate — reading a fixture file, recording a selection, writing the final manifest. Still no payload leakage if conductor stays disciplined.
Return-Summary
Reserved · not exercised yet
Worker returns the payload to the orchestrator, which writes the store. Bounded poisoning — one step, not history. Reserved for cases where the orchestrator must reason about the value mid-flight.

CThe Oscar Fix — Isolation Discipline

THE ORCHESTRATOR NEVER INGESTS PAYLOADS
Five rules · proven across 4 probes · verified by grep at end of every run
  • Workers are subagents — clean context, return only one-line acks. No parent history.
  • The conductor log captures only dispatches, acks, gate transitions, timing. Never the data itself.
  • The transcript is on disk — workers read it via their own bash. Orchestrator holds only the path.
  • Image bytes never leave disk — workers curl -o them locally. Store record carries only localPath.
  • Critique text lives in the store, not the log. Log records only the byte length of the user's input.

DThe EAV Envelope · Every Store Record

// Every line of store.jsonl follows this exact shape.
// `value` is any JSON. `group` + `index` are null for scalars,
// or used together for collections (e.g. 6 explorationImages).
{
  "key":    "<string>",            // the addressable name
  "group":  "<string>" | null,     // for collections
  "index":  <int> | null,           // slot within group
  "value":  <any JSON>,             // the payload — opaque to addressing
  "meta": {
    "step":    "<step-id>",
    "by":      "subagent:<step-id>" | "conductor",
    "ts":      "<ISO-8601>",
    "attempt": <int>,
    "status":  "ok" | "error" | "candidate",
    "source":  "<optional>",         // e.g. "human-stdin", "fixture"
    "note":    "<optional>"
  }
}
