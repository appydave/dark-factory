01
Bulk Content Analysis
Can the conductor stay clean under 12-way parallel writes?
16/16 ✓

What Was Tested

Parallel fan-out: 12 subagents extract distinct content-analysis fields from a single transcript simultaneously. Each worker writes a distinct store key. Orchestrator dispatches all 12 at once and waits for acks.

  • type: all transform · all parallel · distinct stores
  • dispatches: 12 subagents (Agent tool)
  • elapsed: ~37s wall clock
  • prompts: 12 .hbs lifted verbatim from AWB Gen 3

Store Keys Produced

  • mainTopic · keywords · audienceInsights
  • emotionalTriggers · statistics · catchyPhrases
  • ctaPhrases · keyTakeaways · overallTone
  • questionsPosed · searchTerms · usp
Key insight: distinct-key fan-out works. Each worker has a clean context; the conductor log contains 12 dispatches + 12 acks and nothing else. Transcript text never leaked.
workflow: ylo-experiment/workflow-01-analysis.json
outputs: runs/b65/store.jsonl · run-summary.md · conductor-session.log
Probe
02
Scripted Refinement Loop
Does refinement preserve attempt history while keeping orchestrator clean?
21/21 ✓

What Was Tested

Sequential refinement loop with a fixture-driven critique. Plus a new mechanism — outputKeyMap — that lets a single step write two distinct store keys (one for the refined collection, one for the picked-best). Tests both EAV ordering and key-rename semantics.

  • flow: generate → checkpoint(fixture) → refine+outputKeyMap
  • dispatches: 2 subagents + 1 conductor-write
  • elapsed: ~56s
  • fixture: critique-fixture.txt

Store Keys Added

  • shortTitle (seed)
  • generatedTitles attempt:1 (10 titles)
  • titleCritiqueLog
  • generatedTitles attempt:2 (refined)
  • selectedTitles (via outputKeyMap from topTitle)
Key insight: stateless re-dispatch works. The refine subagent reads both the prior attempt and the critique from disk; the conductor never holds either. Ordered subsequence GT1 → CL → GT2 → ST verified.
workflow: ylo-experiment/workflow-02-titles.json
outputs: runs/b65/store.jsonl (appended) · run-summary-titles.md · conductor-session-titles.log
Probe
03
Real Human Gate
Can the orchestrator host a live conversation without polluting the store-fed workers?
21/21 ✓

What Was Tested

Live human-in-the-loop critique via stdin. Conductor renders 10 candidate titles, yields the turn, parses the next user message as critique. Both branches exercised in one run: user typed ok first (accept-path) then a real critique (refine-path).

  • flow: generate → checkpoint(humanGate human-stdin) → refine
  • dispatches: 2 subagents + interactive turn yield
  • store dir: runs/b65-2-5/ (isolated from main b65 store)
  • critique length: 56 bytes typed by user

Both Paths Exercised

  • ACCEPT: typed "ok" → selectedTitles + meta.skippedRefinement:true
  • REFINE: typed critique → titleCritiqueLog source:human-stdin
  • HUMAN-GATE-OPENED + HUMAN-GATE-CLOSED logged
  • critique-length:N recorded (NOT content)
  • append-only preserved both paths as audit history
Key insight: the orchestrator can host an interactive conversation while keeping workers isolated. Only distilled critique moves to the next worker — raw chat stays in conversation history, never in the store-fed payload.
workflow: ylo-experiment/workflow-03-titles-human.json
outputs: runs/b65-2-5/store.jsonl · conductor-session-titles-human.log · run-summary-titles-human.md
side effect: surfaced /goal-vs-human-gate friction → resolved as orthogonal layers (goal is build-time, conductor is runtime)
Probe
04
kie.ai Thumbnails + Title Pairing
Does isolation hold for binary outputs from a long-running external API?
20/22 ⚠

What Was Tested

Five-phase workflow: design briefs → 6 exploration thumbnails (cheap, parallel) → fixture selection → 3 final thumbnails (premium 2K, parallel) → manifest. Tests async external API (POST/poll/download), binary outputs to disk, and the FliThumb title-thumbnail pairing invariant.

  • flow: briefs → 6× parallel kie.ai → fixture → 3× parallel kie.ai → action
  • dispatches: 12 subagents (1 LLM + 11 image-gen)
  • elapsed: ~20 min wall clock
  • cost: ~$0.36 (over $0.20 budget — model substitution)
  • new skill: .claude/skills/image-gen/SKILL.md

Outputs + Invariants

  • 1 designBriefs record (6 briefs, each with title_ref)
  • 6 explorationImages (group:exploration, index:0..5, localPath)
  • 1 selectedExplorationIndices [0,2,4]
  • 3 finalImages (group:final, sourceExplorationIdx + title_ref)
  • 9 JPGs on disk (124KB–1.8MB)
  • 0 image bytes in store · 0 kie response bodies in log
Key insight: pattern works for external APIs and binary outputs. Title pairing held end-to-end — every final's title_ref matches its source exploration's. Pattern is now production-shaped.
workflow: ylo-experiment/workflow-04-thumbnails.json
outputs: runs/b65/thumbs/ (6 exp + 3 final) · thumbnails-manifest.md · kie-models-research.md · conductor-session-thumbnails.log
