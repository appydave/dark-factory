# unverified causal claim — Claims asserted from inference/memory without checking the actual evidence

## Problem Signature

**Symptoms**: After finding two compaction events preceding the handover-writing tool call in a transcript, the assistant stated as fact that the handover was 'a summary of a summary' and that pre-compaction content (phases 1–3) was no longer available to the writer — without checking whether the content the user later quoted was actually present post-compaction.

**Environment**: Dark Factory repo, session-correction conversation; reasoning over `~/.claude/projects/.../627e86a2-....jsonl`.

**Triggering Conditions**: Investigating why a prior session's handover doc was thin; found compaction line indices (1677, 1678, 3176, 3177) before the handover `Write` call (line 5039) and built a plausible narrative on top of that timing alone.

## Root Cause
The explanation was inferred from timing correlation (compactions occurred before the write) rather than verified against the actual content — the user's own 'loose ends' list, which he had pasted into the conversation, came directly from the post-compaction portion of that same prior session, proving the information was present and simply never captured, not lost to compaction.

## Solution
On direct challenge, retracted the compaction claim explicitly and re-investigated: grepped the transcript for actual Skill tool_use calls and found the true root cause (see the separate 'unused skill family' learning) — then corrected the durable memory record to state the retracted claim plus the verified root cause.

```diff-before
**Completeness is the hard part (David, 2026-06-08).** The first 2026-06-08 handover was "piss-weak" — a *summary* that dropped half the loose ends (forks, named-but-unbuilt, awaiting-review renders). A handover must capture **ALL** open threads, not a digest. Fix when one is thin: **mine the raw session transcript** (`~/.claude/projects/<proj>/<id>.jsonl`, via the `abridge` subagent) and rebuild — done 2026-06-08 from `627e86a2`.
```
```diff-after
**ROOT CAUSE = the TOOL, not the prose (David, 2026-06-08).** The thin first handover wasn't careless writing — it was written by a `Write` call **from the live conversation context AFTER two compactions** (verified: session `627e86a2`, compactions at transcript lines 1677 & 3176; handover Write at line 5039)... **Corollary: any handover written from in-conversation memory — especially post-compaction — is structurally untrustworthy.**

**The fix is the MECHANISM:** a handover must be generated by **mining the durable raw transcript**... NEVER hand-written from live context.
```
(Note: this compaction-as-root-cause explanation was itself retracted moments later in the same session — see the follow-up correction — once the user proved the cited information had in fact survived compaction.)

## Prevention
Before asserting a causal explanation as fact — especially one built from correlated evidence like tool-call timing — verify it against the primary, checkable content (e.g. confirm the specific missing information was actually absent from the relevant context window) rather than presenting a plausible-sounding inference as verified. Apply the ✅ verified / ⚠️ inference / ❓ unknown tagging discipline to self-diagnoses, not just claims about the codebase.
